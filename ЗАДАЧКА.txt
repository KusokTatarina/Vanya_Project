Задание:

Спроектировать систему, которая содержит:
Реляционную БД, согласно варианту задания (варианты приведены ниже)
Бекэнд часть
Фронтенд часть 
Спроектировать RestFull API для backend части приложения. При реализации 
должны выполняться правила REST API (https://habr.com/ru/post/447322/). Для 
каждой сущности необходимо реализовать стандартный набор методов (CRUD):
Получение списка по определённой сущности
Получение единичного элемента по уникальному идентификатору
Создание нового элемента сущности
Редактирование элемента сущности
Удаление элемента сущности
В зависимости от языка реализации разрешается использовать готовые 
фреймворки для построения REST, например:
FastAPI - для языка Python (https://fastapi.tiangolo.com/ru/)
JHipster - для JAVA (https://www.jhipster.tech/)
Laravel - для PHP (https://laravel.com/)
Entity Framework - для ASP.NET (https://www.learnentityframeworkcore.com/)
Описать систему в модели C4
C1 и С2 на оценку удовлетворительно
C1, C2 и С3 на оценку хорошо
C1, С2, C3 и С4 на оценку отлично



Спроектируйте БД, содержащую сведения об участниках конкурса бальных  танцев. БД 
должна содержать следующую информацию:  
Справочник «Виды бальных танцев» - {Бальные танцы, Хастл, Исторический 
танец, Латиноамериканский танец и т.п.}
Справочник «Тренеры России по бальным танцам», который содержит ФИО 
тренера и вид бального танца, по которому он тренирует (один тренер может 
вести только один вид танцев)
Информация о коллективе, выступающем на конкурсе содержит название 
коллектива, ФИО тренера из справочника тренеров, год основания коллектива.
Информация об участнике содержит  - фамилия участника,  название 
коллектива, фотография участника, город, класс, баллы, полученные за 
выступление в отдельных программах конкурса (например обязательная 
программа, произвольная программа и т.п.).
